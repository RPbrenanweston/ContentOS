# Plan 02-02: Sales Researcher & Resourcer Agents

## Goal

Build the Sales Researcher and Resourcer agents with domain-tuned prompts, implement hybrid LLM strategy (Claude for reasoning, Ollama for enrichment), and validate end-to-end on sample data.

## Must-haves

1. Sales Researcher agent retrieves signals from Supabase and validates relevance using Claude
2. Sales Researcher filters low-confidence signals and prioritizes hiring indicators
3. Resourcer agent analyzes hiring patterns and identifies trends using hybrid LLM
4. Resourcer agent performs competitive positioning analysis
5. Both agents inherit from base_agent and integrate with orchestrator
6. Prompts tuned for AI security domain (GTM/Engineering roles, threat prevention, hiring patterns)
7. Hybrid LLM working: Claude for complex reasoning, Ollama/Mistral for summarization
8. Agents store structured outputs to Supabase (signals table with validation scores)
9. Unit tests for both agents with sample company data
10. Cost tracking: log Claude API tokens and estimated cost per agent run

## Tasks

### 1. Build Sales Researcher Agent

- Create `agents/sales_researcher.py`:
  - Class: `SalesResearcherAgent` inherits from `BaseAgent`
  - Method: `execute(context)`:
    - Retrieve raw signals for company from Supabase (RSS feeds, CSV enrichment)
    - Filter signals by confidence_level (>= 0.6)
    - Use Claude to validate each signal's relevance:
      - Is this a hiring signal? (job posting, team expansion, funding for hiring)
      - Does it match target ICP? (GTM roles: Sales, Marketing, RevOps; Engineering: Security, Platform)
      - Extract key details: role title, seniority, urgency indicators
    - Score validated signals: 0.0-1.0 scale
    - Return structured output: list of validated signals with scores
  - Method: `_validate_signal_with_claude(signal)`:
    - Calls Claude API with structured prompt
    - Returns validation result + confidence score
  - Store validated signals to `signals` table with `validated_by_agent=true`

### 2. Build Resourcer Agent

- Create `agents/resourcer.py`:
  - Class: `ResourcerAgent` inherits from `BaseAgent`
  - Method: `execute(context)`:
    - Receive validated signals from Researcher (via context)
    - Analyze hiring patterns:
      - Frequency: hiring velocity (roles per quarter)
      - Seniority distribution: IC vs leadership
      - Department focus: GTM vs Engineering ratio
    - Identify trends using Ollama/Mistral:
      - Summarize hiring themes (e.g., "expanding enterprise sales team")
      - Competitive positioning: compare to similar companies in AI security space
    - Use Claude for strategic insights:
      - GTM strategy indicators (product-led vs sales-led)
      - Market positioning (threat prevention vs detection)
      - Urgency signals (rapid expansion, leadership hires)
    - Return structured output: hiring_analysis, trends, competitive_positioning
  - Method: `_analyze_patterns_with_ollama(signals)`:
    - Uses local Ollama/Mistral for summarization
    - Returns hiring themes and frequency analysis
  - Method: `_strategic_insights_with_claude(patterns)`:
    - Uses Claude for high-level strategic reasoning
    - Returns GTM/market positioning insights
  - Store analysis to `companies` table (new columns: hiring_analysis, updated_at)

### 3. Create Domain-Tuned Prompts

- Create `prompts/sales_researcher.txt`:
  - Signal validation prompt template
  - AI security domain context: GTM roles (Sales Engineer, Solutions Architect, CSM), Engineering roles (Security Engineer, Platform Engineer, DevSecOps)
  - Hiring indicators: "expanding", "growing team", "Series A/B", "just closed funding"
  - Examples of high-confidence vs low-confidence signals
  - Output format: JSON with {is_hiring_signal, relevance_score, key_details, reasoning}

- Create `prompts/resourcer.txt`:
  - Hiring pattern analysis prompt
  - Trend identification instructions
  - Competitive positioning framework: compare to peers (Wiz, Lacework, Orca Security)
  - AI security market context: threat prevention vs detection, cloud-native vs legacy
  - Output format: JSON with {hiring_velocity, seniority_distribution, themes, competitive_position, strategic_insights}

- Create `agents/prompts.py`:
  - Load prompts from files
  - Template rendering with Jinja2
  - Prompt versioning (track which prompt version generated each output)

### 4. Implement Hybrid LLM Strategy

- Update `agents/llm_clients.py`:
  - Method: `call_claude(prompt, model="claude-3-5-sonnet-20241022")`:
    - Make API call to Anthropic
    - Track tokens used (input + output)
    - Calculate cost: $3/MTok input, $15/MTok output
    - Log: timestamp, model, tokens, cost
    - Return: response text + metadata
  - Method: `call_ollama(prompt, model="mistral")`:
    - Call local Ollama instance
    - No cost tracking (runs locally)
    - Retry with exponential backoff if Ollama not available
    - Return: response text

- Usage pattern:
  - Sales Researcher: Claude only (high-accuracy validation needed)
  - Resourcer: Ollama for summarization → Claude for strategic insights (hybrid)

### 5. Integrate with Orchestrator

- Update `agents/orchestrator.py`:
  - Import SalesResearcherAgent and ResourcerAgent
  - In `run_pipeline()`:
    - Step 1: Spawn SalesResearcherAgent, execute, store outputs to context
    - Step 2: Spawn ResourcerAgent, pass Researcher output via context, execute
    - Store all agent outputs to Supabase
    - Log execution time and Claude API cost per agent

### 6. Cost Tracking

- Create `agents/cost_tracker.py`:
  - Track Claude API usage per agent run
  - Aggregate costs: per company, per day, per month
  - Target validation: ~$20-30/month for 10 accounts = $2-3 per account/month
  - Method: `log_api_call(agent_name, model, tokens_in, tokens_out, cost)`
  - Method: `get_monthly_cost()` → returns total spent this month
  - Store cost data in Supabase: new table `agent_costs` (date, agent, company_id, tokens, cost)
