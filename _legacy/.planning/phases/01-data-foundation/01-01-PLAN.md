# Plan 01-01: Supabase Schema Design & Setup

## Goal

Establish the complete Supabase database schema with all required tables (companies, prospects, candidates, signals, matches, feedback), relationships, constraints, and audit fields. This creates the data foundation that CSV uploads, RSS feeds, and agents will read/write to.

## Must-haves

1. **Companies table** with unique identifiers (name + domain), enrichment fields (employee_count, funding, revenue, industry), audit metadata (created_at, updated_at, source_of_truth, confidence_level)
2. **Prospects table** (decision makers) with contact fields (name, email, title, seniority, departments), foreign key to companies, audit metadata
3. **Candidates table** (supply-side talent) with detailed tracking (skills, compensation, availability, recruiter_notes, interview_history), independent from prospects
4. **Signals table** linked to both prospects AND companies, with classification (type: HIRING/COMPANY/INDIVIDUAL, tags: hiring_urgency, competitor_signal, expansion_signal), confidence scoring (0-100), source tracking, aging/soft-delete fields (is_active, engagement_status)
5. **Matches table** to track candidate-prospect pairings (prepares for matching engine v2)
6. **Feedback table** for thumbs up/down on signals (trains filtering accuracy over time)
7. **Database migration script** that creates all tables with proper indexes, constraints, and relationships
8. **Row Level Security (RLS) basics** configured for auth (prepare for multi-user access)
9. **Documentation** of schema design, field definitions, and relationships

## Tasks

### 1. Design Core Tables Schema

**Companies Table:**
```sql
- id (uuid, primary key)
- name (text, required)
- domain (text, required, unique)
- industry (text)
- employee_count (integer)
- funding_stage (text)
- total_funding (numeric)
- revenue_range (text)
- headquarters_location (text)
- created_at (timestamptz, default now())
- updated_at (timestamptz, default now())
- source_of_truth (text, e.g., 'csv_upload', 'rss', 'manual')
- confidence_level (integer, 0-100)
- metadata (jsonb, flexible storage for additional fields)

Indexes:
- domain (unique)
- name (for fuzzy search)
```

**Prospects Table:**
```sql
- id (uuid, primary key)
- company_id (uuid, foreign key → companies.id, cascade delete)
- name (text, required)
- email (text)
- phone (text)
- title (text)
- seniority (text, e.g., 'C-Level', 'VP', 'Director', 'Manager')
- departments (text[], array of departments)
- linkedin_url (text)
- location (text)
- created_at (timestamptz, default now())
- updated_at (timestamptz, default now())
- source_of_truth (text)
- confidence_level (integer, 0-100)
- metadata (jsonb)

Indexes:
- company_id (for fast joins)
- email (for deduplication)
- seniority (for filtering decision makers)
```

**Candidates Table:**
```sql
- id (uuid, primary key)
- name (text, required)
- email (text)
- phone (text)
- current_title (text)
- current_company (text)
- skills (text[], array of skills)
- years_experience (integer)
- compensation_expectation (text)
- availability (text, e.g., 'immediate', '2_weeks', '1_month')
- linkedin_url (text)
- resume_url (text)
- recruiter_notes (text)
- interview_history (jsonb, stores array of interview records)
- placement_status (text, e.g., 'active', 'placed', 'inactive')
- created_at (timestamptz, default now())
- updated_at (timestamptz, default now())
- source_of_truth (text)
- metadata (jsonb)

Indexes:
- email (for deduplication)
- skills (GIN index for array search)
- placement_status (for filtering active candidates)
```

**Signals Table:**
```sql
- id (uuid, primary key)
- company_id (uuid, foreign key → companies.id, nullable)
- prospect_id (uuid, foreign key → prospects.id, nullable)
- signal_type (text, required, e.g., 'HIRING', 'COMPANY', 'INDIVIDUAL')
- title (text, required, brief description)
- description (text, full content/summary)
- source (text, e.g., 'techcrunch_rss', 'csv_upload', 'linkedin')
- source_url (text, link to original content)
- tags (text[], e.g., ['hiring_urgency:high', 'competitor_signal', 'expansion_signal'])
- confidence_score (integer, 0-100)
- detected_at (timestamptz, when signal was discovered)
- is_active (boolean, default true, soft-delete for aging)
- engagement_status (text, e.g., 'new', 'contacted', 'deal_won', 'deal_lost')
- engaged_at (timestamptz, nullable, when prospect was contacted)
- created_at (timestamptz, default now())
- updated_at (timestamptz, default now())
- metadata (jsonb, stores additional extraction data)

Indexes:
- company_id (for fast company signal lookups)
- prospect_id (for fast prospect signal lookups)
- signal_type (for filtering by category)
- is_active (for filtering active signals)
- detected_at (for time-based queries)
- engagement_status (for tracking conversions)

Constraints:
- Check: At least one of company_id or prospect_id must be non-null
```

**Matches Table:**
```sql
- id (uuid, primary key)
- candidate_id (uuid, foreign key → candidates.id, cascade delete)
- prospect_id (uuid, foreign key → prospects.id, cascade delete)
- signal_id (uuid, foreign key → signals.id, nullable)
- match_score (integer, 0-100, algorithm-generated)
- match_reason (text, explanation of why matched)
- status (text, e.g., 'suggested', 'presented', 'interviewing', 'placed', 'rejected')
- created_at (timestamptz, default now())
- updated_at (timestamptz, default now())
- metadata (jsonb)

Indexes:
- candidate_id (for candidate-centric queries)
- prospect_id (for prospect-centric queries)
- status (for filtering active matches)
```

**Feedback Table:**
```sql
- id (uuid, primary key)
- signal_id (uuid, foreign key → signals.id, cascade delete)
- user_id (uuid, nullable, future: link to auth.users)
- feedback_type (text, 'thumbs_up' or 'thumbs_down')
- feedback_note (text, optional explanation)
- created_at (timestamptz, default now())

Indexes:
- signal_id (for aggregating feedback per signal)
- feedback_type (for calculating positive/negative ratios)
```
