## Codebase Patterns
<!-- Ralph will populate this section as it discovers patterns -->
- Existing client.ts already has Anthropic (lines 166-205) and OpenAI (lines 206-248) branches — follow that pattern
- OpenAI client creation pattern: `new OpenAI({ apiKey })` — for OpenRouter add `baseURL: 'https://openrouter.ai/api/v1'`
- Provider detection: `model.provider` field from ai_models table registry
- resolveKey() in keys.ts is already generic — uses `process.env[`${provider.toUpperCase()}_API_KEY`]` for managed fallback
- Python client.py is currently Anthropic-only — needs full provider abstraction
- Message formatting differs: Anthropic uses `messages.create()`, OpenAI uses `chat.completions.create()`
- ai_models table provider column is unconstrained TEXT — no enum, just insert 'openrouter'
