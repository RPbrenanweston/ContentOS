## Codebase Patterns
- **Supabase client pattern:** Three client factories in content-os/src/infrastructure/supabase/client.ts — browser (anon), server (access token), service (bypasses RLS). Studio has separate clients at lib/supabase/client.ts and server.ts.
- **Repository pattern:** DAT layer uses typed repos at infrastructure/supabase/repositories/ — each maps DB rows to domain entities.
- **Service DI:** services/container.ts is the dependency injection root — creates all service instances with shared Supabase client.
- **Platform adapter pattern:** infrastructure/distribution/platform-adapter.ts defines the interface. Existing adapters: linkedin, x, bluesky, threads, reddit. All implement getAuthUrl(), exchangeCode(), createPost(), deletePost(), getMetrics(), validateCredentials().
- **API route pattern:** Next.js App Router at src/app/api/. All routes use withApiHandler() wrapper from lib/api-handler.ts.
- **Domain models:** TypeScript interfaces at domain/ with enums.ts centralizing allowed values.
- **Queue:** pg-boss for job processing at infrastructure/queue/pg-boss.ts. Workers at infrastructure/queue/workers.ts.
- **Migration pattern:** Sequential numbered SQL at supabase/migrations/001-014.
- **ai-core integration:** createAIClient(config) from packages/ai-core. ChatParams/ChatResult types match 1:1.
- **Self-hosted Supabase:** Coolify at 204.168.149.15, Kong on port 8000, Postgres on 5432.
- **Vercel:** contentos-app and contentos-studio projects, auto-deploy from GitHub RPbrenanweston/ContentOS.
