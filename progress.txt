## Codebase Patterns
- **Supabase client pattern:** Three client factories in content-os/src/infrastructure/supabase/client.ts — browser (anon), server (access token), service (bypasses RLS). Studio has separate clients at lib/supabase/client.ts and server.ts.
- **Repository pattern:** DAT layer uses typed repos at infrastructure/supabase/repositories/ — each maps DB rows to domain entities.
- **Service DI:** services/container.ts is the dependency injection root — creates all service instances with shared Supabase client.
- **Platform adapter pattern:** infrastructure/distribution/platform-adapter.ts defines the interface. LinkedIn and X adapters implement getAuthUrl(), exchangeCode(), publish(), getMetrics().
- **API route pattern:** Next.js App Router at src/app/api/. All routes use withApiHandler() wrapper and auth middleware.
- **Breadcrumb format:** Line-comment style: // @crumb id / // [AREA] | tags / // why: / // in: out: err: / // hazard: / // edge: -> TYPE / // prompt:
- **Domain models:** TypeScript interfaces at domain/ with enums.ts centralizing allowed values.
- **Queue:** pg-boss for job processing at infrastructure/queue/pg-boss.ts. Workers at infrastructure/queue/workers.ts.
- **Existing tests:** packages/ai-core has 18 TS tests + 6 Python tests. Apps have NO tests yet.
- **Migration pattern:** Sequential numbered SQL files at supabase/migrations/001-013.
- **ai-core integration:** createAIClient(config: {appId, supabaseClient, defaultModel?}) from packages/ai-core/src/client.ts. ChatParams={userId, featureId, messages, model?, maxTokens?, temperature?, tools?}. ChatResult={content, usage:{tokensIn, tokensOut, costUsd}, model, latencyMs}. Types match Content OS facade almost 1:1.
