# Plan 02-01: Autonomy Framework Setup + Orchestrator

## Goal

Set up the Autonomy SDK framework and build the core orchestrator that manages the agent pipeline (Sales Researcher → Resourcer → Copywriter) with shared context via Supabase.

## Must-haves

1. Autonomy SDK installed and configured with proper dependencies
2. Base agent class that all agents inherit from (Supabase context access)
3. Context manager for shared state across agents using Supabase
4. Orchestrator that executes agents in sequence: Researcher → Resourcer → Copywriter
5. Error handling and retry logic for agent failures
6. Orchestrator spawns agents per company account
7. Agent outputs passed as input to next agent in pipeline
8. Configuration for hybrid LLM setup (Claude API + Ollama/Mistral placeholders)
9. Logging infrastructure to track agent execution flow
10. Unit tests for orchestrator basic flow

## Tasks

### 1. Install Autonomy Framework Dependencies

- Add to `requirements.txt`:
  - `anthropic>=0.18.0` (Claude API)
  - `ollama>=0.1.0` (local LLM)
  - `supabase>=2.0.0` (already in Phase 1)
  - `pydantic>=2.0.0` (data validation)
  - `python-dotenv>=1.0.0` (config)
- Create `.env.example` with required keys:
  - `ANTHROPIC_API_KEY`
  - `OLLAMA_BASE_URL` (default: http://localhost:11434)
  - `SUPABASE_URL`
  - `SUPABASE_KEY`

### 2. Build Base Agent Class

- Create `agents/base_agent.py`:
  - Abstract base class for all agents
  - Initialize with company_id, Supabase client, LLM clients (Claude + Ollama)
  - Methods: `execute()`, `validate_input()`, `store_output()`
  - Access to shared context via context_manager
  - Error handling wrapper
  - Execution timing/cost tracking

### 3. Build Context Manager

- Create `agents/context_manager.py`:
  - Manages shared state between agents
  - Loads company data, signals, prospects from Supabase
  - Provides context to agents: company profile, signals, previous agent outputs
  - Stores agent outputs to Supabase (signals table, feedback table)
  - Thread-safe context updates
  - Methods: `get_context(company_id)`, `update_context(agent_name, output)`, `commit()`

### 4. Build Orchestrator

- Create `agents/orchestrator.py`:
  - Class: `AgentOrchestrator`
  - Initialize with list of companies to process
  - Method: `run_pipeline(company_id)`:
    - Spawns Sales Researcher agent
    - Passes Researcher output to Resourcer agent
    - Passes Resourcer output to Copywriter agent
    - Handles agent failures with retry logic (max 3 retries)
    - Logs execution time and cost per agent
    - Stores final outputs to Supabase
  - Method: `run_batch(company_ids)`:
    - Executes pipeline for multiple companies
    - Parallel execution support (up to 3 concurrent)
  - Error aggregation and reporting

### 5. Configure Hybrid LLM Setup

- Create `agents/llm_clients.py`:
  - Initialize Claude client (Anthropic SDK)
  - Initialize Ollama client
  - Factory method: `get_llm_client(model_type)` returns appropriate client
  - Cost tracking for Claude API calls
  - Retry logic for API failures (exponential backoff)

### 6. Create Agent Module Structure

- Create `agents/__init__.py`:
  - Export base classes and orchestrator
  - Version info
- Create placeholder files (to be filled in 02-02 and 02-03):
  - `agents/sales_researcher.py`
  - `agents/resourcer.py`
  - `agents/copywriter.py`

### 7. Add Logging Infrastructure

- Configure structured logging:
  - Log agent start/end with timestamps
  - Log context loading and updates
  - Log LLM API calls (model, tokens, cost)
  - Log errors with full stack traces
  - Write logs to `logs/agents/YYYY-MM-DD.log`

### 8. Write Tests

- Create `tests/test_orchestrator.py`:
  - Test orchestrator initialization
  - Test pipeline execution with mock agents
  - Test error handling and retries
  - Test context passing between agents
  - Test batch processing
