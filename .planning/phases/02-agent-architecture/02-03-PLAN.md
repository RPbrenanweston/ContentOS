# Plan 02-03: Copywriter Agent + End-to-End Testing

## Goal

Build the Copywriter agent to generate email building blocks (signals, trends, relevance framing), integrate all three agents into the orchestrator, test end-to-end on 2 pilot accounts, verify outputs in Supabase, and validate cost targets.

## Must-haves

1. Copywriter agent generates structured email building blocks for each validated signal
2. Email blocks include: signal summary, trend context, relevance framing for ICP
3. Copywriter uses Claude for high-quality, personalized content generation
4. All 3 agents (Researcher → Resourcer → Copywriter) integrated in orchestrator
5. End-to-end pipeline tested on 2 pilot AI security accounts
6. Agent outputs stored in Supabase feedback table for thumbs up/down tracking
7. Cost tracking validates $20-30/month target for 10 accounts (~$2-3 per account)
8. Error handling tested: agent failures, API timeouts, data validation errors
9. Execution logs demonstrate full pipeline trace
10. README documentation for running agents and interpreting outputs

## Tasks

### 1. Build Copywriter Agent

- Create `agents/copywriter.py`:
  - Class: `CopywriterAgent` inherits from `BaseAgent`
  - Method: `execute(context)`:
    - Receive validated signals from Researcher (via context)
    - Receive hiring analysis & trends from Resourcer (via context)
    - For each validated signal, generate email building block:
      - Signal summary: concise 1-2 sentence description
      - Trend context: how this signal fits into broader hiring patterns
      - Relevance framing: why this matters to ICP (GTM leader or Engineering leader)
      - Personalization hooks: company-specific details, competitive insights
    - Use Claude for content generation (high quality, natural language)
    - Return structured output: list of email_blocks (one per signal)
  - Method: `_generate_block_with_claude(signal, trends, company_context)`:
    - Calls Claude API with copywriting prompt
    - Returns structured email block JSON
  - Store email blocks to Supabase: new table `email_blocks` (signal_id, company_id, block_text, created_at, feedback)

### 2. Create Copywriter Prompt

- Create `prompts/copywriter.txt`:
  - Email block generation prompt template
  - Tone: credible, consultative, non-salesy
  - Structure: Signal → Trend → Relevance → Hook
  - AI security domain context: threat landscape, hiring urgency for security roles
  - ICP targeting:
    - For GTM leaders: revenue impact, sales enablement, customer success
    - For Engineering leaders: security posture, compliance, platform scalability
  - Examples of high-quality vs generic email blocks
  - Output format: JSON with {signal_summary, trend_context, relevance_framing, personalization_hook}

### 3. Create Email Blocks Table in Supabase

- Add migration: `supabase/migrations/003_email_blocks.sql`
  - Table: `email_blocks`
    - id (uuid, PK)
    - signal_id (uuid, FK to signals)
    - company_id (uuid, FK to companies)
    - block_text (jsonb): {signal_summary, trend_context, relevance_framing, personalization_hook}
    - feedback (text): null (for thumbs up/down in future phase)
    - created_at (timestamp)
    - updated_at (timestamp)
  - Indexes: company_id, signal_id, created_at
- Run migration to create table

### 4. Integrate Copywriter into Orchestrator

- Update `agents/orchestrator.py`:
  - Import CopywriterAgent
  - In `run_pipeline()`:
    - Step 1: SalesResearcherAgent → validated signals
    - Step 2: ResourcerAgent → hiring analysis, trends
    - Step 3: CopywriterAgent → email building blocks
    - Pass all context sequentially
    - Store final outputs: signals (validated), companies (hiring_analysis), email_blocks
  - Log full pipeline execution: start time, end time, agents executed, errors
  - Return summary: number of signals validated, email blocks generated, total cost

### 5. Create Pilot Account Test Script

- Create `scripts/run_pilot_accounts.py`:
  - Load 2 pilot companies from Supabase (e.g., Wiz, Orca Security)
  - Ensure pilot companies have sample signals in database
  - Run orchestrator pipeline for each company
  - Print execution summary:
    - Signals retrieved, validated, filtered
    - Hiring patterns identified
    - Email blocks generated
    - Claude API cost per company
    - Total execution time
  - Save outputs to `logs/pilot_runs/YYYY-MM-DD-company-name.json`

### 6. Write End-to-End Tests

  - Test email block generation with mock signals and trends

  - Test personalization hooks for GTM vs Engineering ICPs

### 7. Validate Cost Targets

- Run pilot accounts and measure Claude API costs:
  - Target: $2-3 per account per run
  - Breakdown:
    - Sales Researcher: ~$0.50-1.00 (validate 5-10 signals)
    - Resourcer: ~$0.50-1.00 (strategic insights)
    - Copywriter: ~$1.00-1.50 (generate 3-5 email blocks)
  - Total per account: ~$2-3
  - Monthly for 10 accounts (daily runs): $20-30/month ✓
- If costs exceed target:
  - Reduce Claude calls (batch signal validation)
  - Use Ollama more aggressively (Copywriter drafts → Claude refinement)
  - Optimize prompt lengths

### 8. Test Error Handling

- Simulate failure scenarios:
  - Claude API rate limit hit → verify exponential backoff retry
  - Ollama service down → verify graceful degradation (skip trend summarization)
  - Invalid signal data → verify validation catches errors
  - Supabase timeout → verify retry logic
- Verify orchestrator logs all errors with context
- Verify pipeline doesn't crash, returns partial results
