# Plan 03-01: Feedback UI + Storage Layer

## Goal

Build the feedback capture mechanism that allows users to rate agent outputs (thumbs up/down) and store feedback in Supabase for learning loop analysis.

## Must-haves

1. **Feedback UI Component**
   - Thumbs up/down buttons rendered on each signal card
   - Visual state changes (selected/unselected)
   - Disabled state after feedback submitted (prevent duplicate votes)
   - Accessible (keyboard navigation, ARIA labels)

2. **Feedback Storage Schema**
   - `feedback` table in Supabase with fields:
     - `id` (UUID, primary key)
     - `signal_id` (UUID, foreign key to signals table)
     - `agent_type` (enum: 'researcher', 'resourcer', 'copywriter')
     - `agent_output` (JSONB, stores the full agent response)
     - `user_rating` (integer: 1 for thumbs up, -1 for thumbs down)
     - `feedback_text` (text, optional user comment)
     - `prompt_version` (text, tracks which prompt version generated this output)
     - `created_at` (timestamp)
     - `user_id` (UUID, nullable for MVP)
   - Indexes on `signal_id`, `agent_type`, `created_at`, `prompt_version`
   - Foreign key constraints with cascade rules

3. **Feedback Capture Service**
   - TypeScript service layer for feedback operations
   - `submitFeedback(signalId, agentType, rating, output, promptVersion)` function
   - `getFeedbackBySignal(signalId)` function
   - `getFeedbackByAgent(agentType, dateRange)` function
   - Error handling and validation

4. **API Endpoints**
   - POST `/api/feedback` - Submit new feedback
   - GET `/api/feedback?signal_id={id}` - Retrieve feedback for a signal
   - GET `/api/feedback?agent_type={type}&start_date={date}&end_date={date}` - Retrieve feedback by agent and date range

5. **Integration with Existing Components**
   - Feedback widget embedded in signal display cards
   - Agent outputs stored with prompt version metadata
   - Real-time feedback state updates (optimistic UI)

## Tasks

### Task 1: Database Schema

**File:** `supabase/migrations/003_feedback_schema.sql`

Create migration file with:
```sql
-- Feedback table for agent learning loop
CREATE TABLE IF NOT EXISTS feedback (
  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  signal_id UUID NOT NULL REFERENCES signals(id) ON DELETE CASCADE,
  agent_type TEXT NOT NULL CHECK (agent_type IN ('researcher', 'resourcer', 'copywriter')),
  agent_output JSONB NOT NULL,
  user_rating INTEGER NOT NULL CHECK (user_rating IN (-1, 1)),
  feedback_text TEXT,
  prompt_version TEXT NOT NULL,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  user_id UUID,

  -- Prevent duplicate feedback per user/signal/agent
  UNIQUE(signal_id, agent_type, user_id)
);

-- Indexes for performance
CREATE INDEX idx_feedback_signal_id ON feedback(signal_id);
CREATE INDEX idx_feedback_agent_type ON feedback(agent_type);
CREATE INDEX idx_feedback_created_at ON feedback(created_at DESC);
CREATE INDEX idx_feedback_prompt_version ON feedback(prompt_version);
CREATE INDEX idx_feedback_rating ON feedback(user_rating);

-- Add prompt_version tracking to signals table
ALTER TABLE signals ADD COLUMN IF NOT EXISTS researcher_prompt_version TEXT DEFAULT 'v1.0';
ALTER TABLE signals ADD COLUMN IF NOT EXISTS resourcer_prompt_version TEXT DEFAULT 'v1.0';
ALTER TABLE signals ADD COLUMN IF NOT EXISTS copywriter_prompt_version TEXT DEFAULT 'v1.0';
```

Run migration against Supabase instance.
